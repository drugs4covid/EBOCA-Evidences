EBOCA Evidences: Description back to ToC

Figure 1 provides a general overview of the EBOCA Evidences Ontology. This representation follows the [Chowlk notation](https://chowlk.linkeddata.es/notation.html).

The EBOCA Evidences Ontology aims to describe provenance and metadata for evidences of associations among concepts. It reuses classes and properties from several ontologies: Semanticscience Integrated Ontology (SIO), Evidence & Conclusion Ontology (ECO), Friend Of A Friend (FOAF), DCMI Metadata Terms, Provenance, Authoring and Versioning (PAV), and the SPAR Ontologies FRBR-aligned Bibliographic Ontology (FaBiO), Document Components Ontology (DoCO).

The associations between concepts are represented with the class sio:SIO_000897 "association". This association can be annotated with its creator (pav:createdBy), that is a foaf:Agent; and the software (dc:Software) it was created with (pav:createdWith). This association is linked to its correspondent Evidence with sio:SIO_000772 "has evidence".

The class Evidence is a subclass of obo:ECO_0000000 "evidence". Two subclasses of "evidence" are highlighted, "computational inference" and "documented statement evidence". Evidences can be annotated optionally with a confidence score (score), date of creation (pav:createdOn) and update (pav:lasUpdatedOn), and version (pav:version). The provenance of the evidence can be indicated with pav:derivedFrom.

For that reason, the class doco:Paragraph is reused. To add more information to where the paragraphs come form and where can they be found, the class fabio:Expression is reused.
